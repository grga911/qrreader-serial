# Security Policy

## Supported versions

Security fixes are applied to the latest release on `main` and to the most
recent tagged GitHub Release when practical.

| Version | Supported |
|---------|-----------|
| Latest release / `main` | Yes |
| Older releases | Best effort |

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security problems.

Prefer one of these private channels:

1. **GitHub Security Advisories** (preferred):  
   <https://github.com/grga911/qrreader-serial/security/advisories/new>
2. **Email**: [grgicnemanja@gmail.com](mailto:grgicnemanja@gmail.com)

Include as much detail as you can:

- Affected version / commit
- Environment (OS, install method)
- Steps to reproduce
- Impact (e.g. clipboard data exposure, privilege escalation, unsafe install script behavior)
- Any suggested fix

## What to expect

- Acknowledgement when the report is received
- An initial assessment and, if confirmed, a fix or mitigation plan
- Credit in the advisory or release notes if you want it (optional)

We ask that you give us a reasonable window to fix and release before any
public disclosure.
